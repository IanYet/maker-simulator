# 数据层级

## 数据类型

| 数据 | 说明 | 写入时机 |
| --- | --- | --- |
| 默认数据 | 游戏原始配置，包含全部角色、效果、效果组合、候选池、事件 | 编辑器或内容构建阶段 |
| 玩家存档 | 从默认数据生成，保存局外解锁、长期成长、累计进度 | 创建存档、结束一局、局外结算 |
| 局内动态数据 | 从玩家存档生成，保存当前单局状态 | 开始一局、单局进行中、每回合快照 |

## 生成关系

| 流程 | 结果 |
| --- | --- |
| 默认数据 + 开始前选项 | 玩家存档 |
| 玩家存档 | 局内动态数据 |
| 局内动态数据中的局外保留项 | 玩家存档 |

## 顶层结构

默认数据、玩家存档、局内动态数据使用同一顶层结构。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `meta` | object | 数据元信息 |
| `character` | object | 当前角色 |
| `effectKinds` | array | 当前内容支持的效果类型声明 |
| `effects` | array | 效果实例与效果定义状态 |
| `effectCombos` | array | 效果组合规则 |
| `pools` | array | 效果或事件的候选筛选与抽取规则 |
| `events` | array | 事件定义与事件状态 |

## meta 字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | string | 数据 ID |
| `version` | string | 内容版本 |
| `turn` | number | 当前回合；默认数据与玩家存档通常为 0 |
| `seed` | string/null | 当前随机种子 |
| `step` | string | 当前回合步骤，仅局内动态数据需要 |
| `runs` | number | 已开始局数，仅玩家存档需要 |

## 数据写入规则

1. 默认数据原则上只读。
2. 普通事件、效果触发、节点跳转写入局内动态数据。
3. 局外解锁、长期升级、跨局累计值写入玩家存档。
4. 每个动作必须声明数据作用域；未声明时默认为局内动态数据。
5. 局内临时效果不写回玩家存档，除非动作作用域明确为玩家存档。
