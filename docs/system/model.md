# 游戏模型文档

本模型描述一个回合制、事件驱动、效果构建的 Roguelike 系统。系统数据由 JSON 表达，内容换皮与数值调整通过数据完成。

## 核心原则

1. 数据分为默认数据、玩家存档、局内动态数据。
2. 三层数据结构保持一致，主要差异是字段取值。
3. 角色只保存属性与属性值。
4. 非叙事状态归入效果。
5. 叙事内容归入事件。
6. 事件内部是有向图，由节点和节点跳转组成。
7. 效果组合是独立规则集合。
8. 候选池只定义候选筛选与抽取规则，抽取后的行为由调用动作定义。
9. 解锁、层数、计数、标签等状态保存在对应效果或事件字段上。
10. 暂不支持单回合内撤销、重做、回溯。
11. 每回合结束保存一份完整局内动态数据快照。
