# 候选池

候选池用于从效果或事件集合中筛选并随机抽取对象。候选池只定义候选与抽取规则，不保存抽取后的动作，也不直接修改数据。

## Pool 字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | string | 候选池 ID |
| `selector` | object | 候选集合选择器；`target` 决定抽取效果或事件 |
| `count` | any | 默认抽取数量，支持值表达式 |
| `unique` | boolean | 同一次抽取是否禁止重复候选 |
| `weight` | any | 单个候选的抽取权重，支持值表达式；未声明时为 1 |

## 抽取规则

1. 使用 `selector` 筛选效果或事件。
2. 排除未解锁或已经出现的候选。
3. 判断候选自身的 `appear.conditions`，再执行 `appear.chance` 判定。
4. 在候选上下文中计算 `weight`。
5. 使用局内随机种子抽取 `count` 个候选。
6. `unique=true` 时采用不放回抽取；候选不足时返回全部剩余候选。

权重小于等于 0 的候选不参与抽取。相同数据和随机种子必须产生相同抽取结果。
抽取数量求值后必须是大于等于 0 的整数。

## 候选上下文

计算 `weight` 时，值表达式可以通过 `$candidate.*` 读取当前候选对象字段。候选上下文只在候选池内部存在。

## 调用规则

候选池由 `draw_pool` 动作调用。候选池可以被不同效果、事件节点或效果组合复用，抽取后的数据修改由每次调用自己的 `onDraw` 定义；抽取为空时由 `onEmpty` 处理。
